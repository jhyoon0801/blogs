Spring MVC 기초 - 1

Spring MVC 기초 - 1 Spring MVC에 관련한 기초 내용을 정리해 나가자. web application 개발 관련 기본 정리 web server란? HTTP 프로토콜을 사용하는 클라이언트로부터의 요청을 받아서, HTML(그림, CSS, 자바스트립트 등의 리소스를 포함한) 문서와 같은 리소스를 반환하는 컴퓨터 프로그램이다. web application이란? 정적 데이터(정적 HTML 문서 등)가 아니라, 각각의 요청에 따라 동적 데이터(동적 HTML문서 등)를 생성하는 프로그램이며, 이 프로그램의 동작은 web server의 호출에 의해 수행된다. web application은 web server와는 별개의 프로세스로도 동작할 수 있고, 동일한 프로세스로도 동작할 수 있다.

summary to use git

Git 사용 방법 정리 Git의 기초 내용 정리와 사용하다 자주 까먹는 내용을 정리해 보자 기존 VCS와 차이점 기존 VCS와 Git의 차이점은 다음과 같다. CVSC CVS, Subversion, Perforce, Bazaar 등의 시스템은 파일의 집합으로정보를 관리하고, 각 파일의 변화를 다음 그림 처럼 시간순으로 관리한다 Git Git은 CVCS와는 다르게 파일 시스템의 Snapshot과 같은 데이터를 저장한다( 크기가 아주 작음 ) Git은 파일이 존재하는 그 순간을 중요하게 여기며, 파일이 달라지지 않으면 성능을 위해 파일을 새롭게 저장하지 않고, 단지 이전 상태의 파일에 대한 링크만 저장한다 Branches Git은 다른 VCS와 비교하여 Branches에 대한 강점이 많다.
